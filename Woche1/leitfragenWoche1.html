Das Hypertext Transfer Protocol (HTTP)

  1. Warum sind HTTP/2 und HTTP/3 eigentlich Binärformate, Klartext ist doch so viel einfacher zu debuggen?

    Antwort: Das Binärformat erhöht die Effizienz, da der Klartext nicht erst interpretiert werden muss. Daher ist das Wechseln auf ein höheres Protokoll nur als Option im Header beim Versenden von Daten gegeben.

  2. Was bedeutet der Fehlercode 400? Versuchen Sie z.B. mithilfe von Netcat auf dem Google Webserver den Fehler zu provozieren.

    Antwort: Der Fehlercode "400: Bad Request" gibt an, das etwas mit dem vom Client verschickten Request nicht stimmt, z.B. fehlerhafte Syntax, ungültige HTTP-Action, etc.

  3. Wir haben gemerkt, dass wir in einer Webanwendung unserer Bank die Kontonummer in der Anfrage stehen haben, also zum Beispiel bekommen wir unter https://mybank.app/kd/144255399/stand den Kontostand unseres Kontos mit der Nummer 144255399 angezeigt. Wir beschließen mal zu schauen, was bei anderen Kontonummern passiert. Natürlich ist unsere Bank nicht so dumm fremde Informationen so einfach rauszurücken und die Anwendung erzeugt natürlich einen Fehler. Welcher HTTP Statuscode ist hier angebracht?

    Antwort: Der Statuscode "403: Forbidden" wäre hier angebracht, da der Client, der auf diese Daten zugreifen will, in diesem Fall nicht die nötigen Berechtigungen hat (ein Admin z.B. dürfte auf diese Informationen zugreifen).

  4. Führen Sie mit Netcat einen HEAD Request auf rheinjug.de durch. Die Anfrage können Sie aus dem Beispiel im Wochenblatt übernehmen, ersetzen Sie GET durch HEAD. Vergleichen Sie das Resultat des GET und des HEAD Requests.

    Antwort:HEAD gibt nur den Header wieder. GET liefert die Antwort als HTML Seite. Eingabe 1. nc -v rheinjug.de 80 2. GET bzw HEAD und dann /foo/bar?test=1 HTTP/1.1

  5. Wir haben auf GitHub ein privates Repository mit der URL https://github.com/hhu-propra2/klausur. Da Sie nicht in der berechtigten Benutzergruppe sind, bekommen Sie einen Fehler, wenn Sie auf die URL zugreifen. Was denken Sie über den Statuscode, den GitHub benutzt hat?

  6. Versuchen Sie mit Netcat einmal einen POST Request eines Browsers mitzuschneiden. Experimentieren Sie auch mit dem Encoding.

    Antwort: Mit HEAD bekommen wir nur die Metadaten der Response, nicht die eigentliche Response zurückgeliefert. Mit GET bekommen wir beides.
